## Why This Source Matters

Protocols can describe messages and tools, but these sources describe the inner control mechanisms that make an LLM agent act over time. They distinguish reasoning-action interleaving, environment feedback, verbal reinforcement, self-feedback, tree search, value-guided planning, and state/goal reflection.

## Validation Or Constraint Ideas

| constraint | applies_to | source evidence | validation style | risk |
|---|---|---|---|---|
| Agent loop representation should separate action, observation, feedback, and reflection. | ReasonActLoop | ReAct, Reflexion, Self-Refine | structural schema | Frameworks may collapse these into trace events. |
| Reflection memory should carry provenance and scope. | VerbalReinforcementMemory | Reflexion | semantic constraint | Self-generated lessons can be wrong or stale. |
| Value-guided search should record budget, evaluator source, and stopping condition when exposed. | SearchTree, ValueFunction | LATS, RAFA | structural plus provenance | Many systems hide these internals. |
| Feedback source should be explicit. | FeedbackSignal | Reflexion, LATS, RAFA | enum/profile | External, heuristic, self, and evaluator feedback differ in trust. |
| Goal-state reflection should distinguish current observed state from desired goal state. | GoalStateReflection | ReflAct | structural schema | Colliding with BDI Goal terminology. |

## Schema Implications

Facts only. No final schema decisions.

- Tool use alone is insufficient for modeling agent mechanisms; action, observation, feedback, and reflection are separate candidates.
- Feedback must be typed by source and trust level because self-feedback, environment feedback, evaluator feedback, and heuristic feedback have different reliability.
- Reflection memory should be treated as generated evidence with provenance and review status, not as truth.
- Search/planning mechanisms require optional value, budget, and stopping metadata when they are represented.
- ReflAct and RAFA strengthen the case for distinguishing current state, planned future trajectory, and goal state.

## Contradictions And Open Questions

- Reflection improves adaptation in several sources, but it can also preserve self-generated errors. Phase 0C needs a trust/provenance model for reflective memory.
- ReAct treats reasoning traces as visible and interpretable, but production systems may expose only summaries or trace events.
- LATS and RAFA make planning/search explicit; many framework agents hide the controller behind an SDK runner. The schema should support both explicit and opaque controllers.
- ReflAct suggests state/goal grounding can outperform added reflection modules, raising a question about whether reflection is a module, a loop phase, or a backbone variant.
